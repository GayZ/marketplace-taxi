<div align="center">*Структура запроса соответствует [[Протоколу|Протокол]].*</div>

----

h1. Запрос

|*method*|Orders.getStatus|
|*version*|1.0|
|*params*|object Params|

h3. object Params

|_.Атрибут|_.[[Тип|Типы JSON атрибутов]]|_.Обязательность|_.Описание|
|orderId|number|обязательный|Идентификатор заказа|

h2. Пример

<pre>
{
  "method": "Orders.getStatus",
  "version": "1.0",
  "params": {
    "orderId": 20333
  }
}
</pre>

----

h1. Ответ

|*result*|object Order|

h3. object Order

|_.Атрибут|_.[[Тип|Типы JSON атрибутов]]|_.Обязательность|_.Описание|
|status|number OrderStatus|обязательный|Статус заказа|
|cancellationCause|object CancellationCause|опциональный|Причина отмены (если status=6)|

h3. number OrderStatus

|_.Значение|_.Описание|
|1|Не распределен|
|7|Зарезервирован водителем (только если предварительный)|
|2|Назначен водитель|
|3|Водитель по адресу (ожидает клиента)|
|4|Выполняется|
|5|Успешно завершен|
|6|Отменен|

h3. object CancellationCause

|_.Атрибут|_.[[Тип|Типы JSON атрибутов]]|_.Обязательность|_.Описание|
|kind|number CancellationCauseKind|обязательный|Тип причины|
|comment|string|опциональный|Комментарий|

h3. number CancellationCauseKind

|_.Значение|_.Описание|
|1|По желанию клиента|
|2|По вине диспетчерской|
|3|По вине водителя|
|4|Технические проблемы|
|5|Неадекватный клиент|
|6|Клиент уже уехал|
|7|Клиент не берет трубку|
|8|Автоматическая отмена|

h2. Пример

<pre>
{
  "result": {
    "status": 6,
    "cancellationCause": {
      "kind": 5,
      "comment": "рассказывал несмешные анекдоты"
    }
  }
}
</pre>
